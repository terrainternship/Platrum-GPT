## Об инструменте
Доски - способ организации работы с задачами команды над общим проектом. В досках можно наглядно представить процесс выполнения задач по колонкам, где каждая колонка - это этап работы над задачей.
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73003645803/original/GE70i635qdGkchJtrspRxdwVTSmqfH8jHg.png?1646060899
В Platrum можно создавать неограниченное количество досок, как персональных, так и командных. Вы в любой момент сможете переключаться между ними с помощью списка, а самые часто используемые можно добавить в избранное.

## Создание и настройка доски
Создавать доски могут все сотрудники в проекте, независимо от их доступов. Можно создавать новые доски с нуля каждый раз, когда требуется организовать работу над новым проектом, либо заранее заготовить несколько шаблонов досок с задачами и создавать новые доски на их основе.

### Добавление своих полей на доске
Во всех задачах Platrum есть определенный набор полей по умолчанию. Но помимо них есть возможность создать свои собственные, которые требуются вам в работе с задачами.
Для этого нужно открыть настройки доски по клику на шестеренку и выбрать пункт "Настройка полей":
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013804117/original/_sqPCiB3j9qpBT4JOIySDg4pjmfPIxsAFQ.png?1655980858
Справа откроется меню, где можно будет добавить свое поле:
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013804629/original/VDZxQUKFnXc6PiZdoBi5uB2s33C9xUBrTQ.png?1655981240
У каждого поля можно задать свойства:
- Название - обязательно
- Тип - отвечает за то, какое содержимое должно быть в этом поле. Так, если вы выберете тип "Число", в поле нельзя будет ввести текст. При выборе типа "Список", в поле нельзя будет ничего вводить, а только выбирать из заранее заданных значений. Список значений у поля можно будет задать в поле, которое  появится при выборе этого типа.
- Значение по умолчанию - можно заполнить, если поле как правило имеет одно и то же значение. Например, у поля Приоритет можно выбрать значение по умолчанию "Средний", если задачи с низким и высоким приоритетом - исключение.
- Скрытое - если установить это свойство, поле не будет видно сотрудникам, работающим с доской, оно будет доступно только администратору доски. Это может быть полезно, если есть информация, которую нельзя раскрывать сотрудникам, но она нужна вам для администрирования. Например, это может быть сумма, которую исполнитель получит за выполненную задачу или конфиденциальная информация о заказчике.
- Обязательное - если на доске будет обязательное поле, то на нее нельзя будет поставить задачу, не заполнив его.
- Наследовать значения - это полезно для работы с чеклистами в задачах. Все поля, у которых наследуются значения, будут заполняться в чеклистах автоматически тем же значением, что указано у родительской задачи.
- Сортировка - отвечает за то, в каком порядке поле будет расположено в карточке задачи. Чем меньше число, тем выше задача будет в списке.
- Системное название - нужно для идентификации этого поля и формируется автоматически, как правило, не требует изменений.
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013806046/original/Q2RzWsu4hFuB-FUT8vkCTmKr_9be1gyaqg.png?1655982181
Поля можно создать для каждой доски свои. Так, например, на досках юридической службы и бухгалтерии будет свой уникальный набор полей, специфичный для деятельности каждого отдела.
Если же есть общие поля, свойственные всем доскам, например, приоритет задачи, его можно использовать повторно. При создании нового поля достаточно начать ввод названия и вам будет предложено выбрать из уже существующих полей с таким же названием:
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013806306/original/0wTm8KibxhR_9YKwApPxXj-T8Amh1WnbVw.png?1655982373

### Приглашение участников и настройка доступов
Чтобы выдать доступы к доске, нужно открыть настройки доски и выбрать “Управление доступами”
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013809053/original/ZWhw6i4IFAXZlsazgdECsBMgPVZ1e_dTRQ.png?1655984015
Как только доска создана, она будет доступна всем руководителям вашей вертикали, а также сотрудникам с доступом “Администрирование всех задач”.
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013810006/original/bf_BPaCu-2kKScYmvgDXuGp_D09gWFg79w.png?1655984465
Это системные доступы, их нельзя изменить или отредактировать, они будут меняться только с изменениями на орг.структуре. Полностью приватную доску в Platrum сделать нельзя, поскольку подразумевается, что это рабочее пространство, часть инфраструктуры компании и руководители должны понимать, какую работу ведут подчиненные.
Роли - это подсказка, кем приходится данный сотрудник создателю доски, чтобы было понятно, почему он имеет к ней доступ:
- Администратор - это владелец проекта и сотрудники, у которых есть доступ "Администрирование задач"
- Руководитель владельца - это сотрудник, стоящий выше  по должности в одной вертикали с владельцем доски.
- Владелец - это тот, кто создал доску, он может быть только один.
В дополнение к системным доступам вы можете дать доступ и другим сотрудникам проекта. Доступы можно выдавать двумя способами:
- конкретному сотруднику - в этом случае при его увольнении следующий сотрудник на той же должности уже не будет иметь доступа к доске
- для должности  - в этом случае независимо от того, кто занимает должность, он будет всегда иметь доступ к доске. При таком доступе можно выбрать, чтобы доступ был и у всех подчиненных данной должности
 (https://secure.helpscout.net/docs/5e70b5062c7d3a7e9ae94a29/article/5f4f6db92c7d3a352e91b044/#manage)
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013810682/original/hDKKXAY8-wxsKxdie3it2mWhVbfHOMtXUw.png?1655984826
Доступы могут быть на:
- Просмотр доски - сотрудник сможет только видеть задачи, но не сможет их редактировать, перемещать между колонками;
- Просмотр +создание задач - с таким доступом сотрудники смогут создавать задачи, просматривать задачи всех участников доски, но редактировать только те, которые создал он сам;
- Работа с доской - дает возможность видеть и редактировать все задачи на доске, но без доступа к настройкам самой доски.

### Создание доски по шаблону
Каждый сотрудник, добавленный в проект, может создавать свои доски и шаблоны. В текущей версии шаблоны доступны только их создателю. Позже шаблоны будут доработаны, чтобы к ним также можно было настраивать свои доступы.
Создать новый шаблон можно из списка досок
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013801867/original/pgoXuOeFDeTaZjwEZ-6rxgkkiUmw0Hm3bw.png?1655979406
Пока шаблон нельзя сделать на основе уже существующей доски, его нужно наполнить с нуля. В будущем доски будут доработаны, чтобы можно было создать шаблон на основании уже имеющейся доски.
Все созданные шаблоны будут находиться в том же списке досок в группе "Шаблоны".
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013802115/original/DpkGP-mH78gwyMCYAwtAgbufgb2qJ0gvVA.png?1655979618
Все карточки задач в шаблонах не являются реальными задачами - они не будут показываться у сотрудников или в списке задач при фильтрации. Карточки задач из шаблонов станут реальными задачами, когда на основании шаблона будет создана доска.
После того, как у вас появился хотя бы один шаблон, при создании доски можно будет выбрать шаблон, по которому нужно ее наполнить.

## Как строится работа с доской
На доске вы можете визуализировать процесс движения задачи по этапам работы. Каждая колонка на доске - это этап работ.
Передвигая карточку задачи из списка в список вы обозначаете ее переход от этапа к этапа. При этом вы всегда можете быстро оценить общее положение дел в данном процессе, увидеть узкие места за счет наглядного отображения всех этапов процесса в одном месте.

#### Настройки колонок
Колонки доски - это панели, в которых группируются разные задачи. Каждая колонка имеет свое название, может быть по-своему отсортирована и сгруппирована, имеет свой цвет.
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73013816698/original/xUudDAUaeYuY6kAu4sVAFHqKa0Xcf7iGvQ.png?1655987166
Для настройки колонки нужно нажать на кнопку с троеточием справа от края колонки:
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73003645802/original/CBTd2AdcD5Ka69puu8VL4x8QCGuiHBpIhA.png?1646060899
В настройках можно изменить:
- Название колонки
- Группировку - в этом случае задачи разбиваются на блоки внутри списка в зависимости от типа группировки
- Сортировку - порядок всех задач внутри списка
- Цвет - можно выбрать из заготовленных, можно указать свой
- Показывать или нет - позволит скрыть неактуальные списки. В отличие от удаления можно будет вернуться к нему в будущем
- Поля карточки задачи - от этого зависит, как задача будет отображаться внутри списка
Любой список можно скрыть с доски, переключив чекбокс "Показывать" в неактивное состояние. Чтобы вернуть его обратно, в настройках доски можно включить переключатель "Показать скрытые".

#### Работа с задачами и колонками
Основной принцип работ с задачами в списках - их простой перенос из списка в список для более удобного восприятия статуса задачи или этапа ее решения. Перенос работает по принципу drag’n’drop (захватить мышью и переместить).
По такому же принципу можно поменять местами и сами списки.
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73003645816/original/IklQ8qWNdFAgpExTcKOhNkpf2_GUzPMpYA.png?1646060903
В любом списке можно открыть карточку задачи во всплавающем окне и отредактировать ее:
- Поменять название
- Добавить описание
- Установить сроки выполнения
- Пометить выполненной
- Удалить
https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/73003645812/original/ecZ4ug6jj_5kVUQYPSpca_LBTSH-OJcncA.png?1646060902
Если нужно оставить комментарий в задаче или посмотреть ее более развернуто, во всплывающем окне по нажатию на пиктограмму карандаша можно открыть окно редактирования задачи. Там, помимо вышеперечисленного можно будет:
- Описать продукт
- Изменить исполнителя
- Задать категорию
- Указать планируемое и фактическое время на выполнение задачи
- Оставить комментарий
- Клонировать задачу
 (https://secure.helpscout.net/docs/5e70b5062c7d3a7e9ae94a29/article/5f4f6db92c7d3a352e91b044/#manage)

## FAQ
Здесь собраны частые вопросы от клиентов и ответы на них:

### Есть доска в задачах, в ней работает несколько человек, что будет, если владелец (создатель) доски будет уволен?
При увольнении владельца доски (Удалении его из проекта), доска останется, и у администраторов доски, руководителя владельца доски или владельца проекта останется к ней доступ.
